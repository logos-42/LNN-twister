# 黎曼投影实现分析

## 问题

> 黎曼投影的实现过程里面，没有出现约束情况，而产生了梯度保证，逻辑上，是在几何约束下完成的才对？

## 当前实现

### 投影操作
```python
def project_to_tangent(self, v: torch.Tensor, p: torch.Tensor) -> torch.Tensor:
    n = F.normalize(p, dim=0)        # 法向量 = 参数归一化
    inner = (v * n).sum()            # 梯度在法向的分量
    return v - inner * n             # 减去法向分量 = 切空间投影
```

### 收缩操作
```python
def retract(self):
    retracted = self.geometry.exp_map(old_p, -step)
    p.data.copy_(retracted)
```

## 为什么"没有出现约束"？

### 1. 流形定义太弱

当前法向量 `n = p / ||p||`，约束是**参数保持在以原点为中心的球面上**。

但问题是：
- 权重矩阵 W ∈ ℝ^(463×128) = 59,264 维
- 法向量只定义了 **1 个约束方向**
- 切空间维度 = 59,263（几乎等于全空间）

**结果**：投影只去掉了梯度在径向的 1 个分量，剩下 59,263 个自由度几乎不受约束。

### 2. 流形几何与参数维度不匹配

`ManifoldGeometry` 的法向量计算：
```python
def _get_manifold_normal(self, p: torch.Tensor):
    theta = torch.linspace(0, 2π, N)  # N 是神经元数, 不是参数维度
    n = stack([sin(θ), cos(θ), sin(θ/2)])  # 只有 3 维!
```

这个法向量是为**神经元状态**设计的（3 维流形），不是为**权重矩阵**设计的。
当把它用于 59,264 维权重时，几何意义就丢失了。

### 3. 实际效果

| 操作 | 数学 | 效果 |
|------|------|------|
| 投影 | g' = g - <g,n>n | 去掉径向分量 |
| 约束强度 | 1/59264 ≈ 0.0017% | 极弱 |
| 梯度范数变化 | ~0.1% | 几乎不变 |

**所以梯度保证不是来自几何约束，而是来自**：
- `tanh` 的有界性（输出 ∈ [-1,1]）
- `-z` 衰减项（自稳定）
- `τ` 归一化（变化率有界）
- `orthogonal(gain=0.5)` 初始化（特征值 < 1）

这些是**液态网络架构本身**的性质，不是黎曼投影的贡献。

## 真正的几何约束应该是什么

### 方案 A: Stiefel 流形约束
```
W ∈ St(n,p) = {W ∈ ℝ^(n×p) | W^T W = I}
投影: Proj_W(G) = G - W·sym(W^T·G)
```
约束权重矩阵的列正交，防止共线性导致的梯度爆炸。

### 方案 B: 谱范数约束
```
||W||_2 ≤ σ_max
投影: W ← σ_max · U·V^T  (SVD 截断)
```
直接限制最大奇异值，从根本上消除梯度爆炸。

### 方案 C: 莫比乌斯流形约束（当前意图）
```
W_ij = f(θ_i, φ_j, Θ)  # 权重由流形坐标和扭转张量决定
```
权重不是自由参数，而是流形上的函数值。这才是真正的几何约束。

## 结论

当前实现是**黎曼优化的骨架**（投影+收缩），但**几何定义不够强**。

梯度有界主要来自液态网络架构，黎曼投影的贡献微乎其微（~0.1%）。

要实现真正的几何约束生长，需要：
1. 重新定义流形：权重 = 流形坐标的函数
2. 优化在流形坐标空间进行，不是直接在权重空间
3. 约束强度 = 流形曲率，不是归一化方向
