## 📋 概述

卡帕斯循环系统 (Karpov Cycle) 是一个**基于 Ollama 本地 LLM 的自动化迭代改进系统**。

### 核心流程

```
┌─────────────┐
│  1. 分析    │ ← 代码分析
└──────┬──────┘
       │
       ▼
┌─────────────┐
│  2. 建议    │ ← LLM 生成改进建议
└──────┬──────┘
       │
       ▼
┌─────────────┐
│  3. 改进    │ ← 生成代码改进
└──────┬──────┘
       │
       ▼
┌─────────────┐
│  4. 测试    │ ← 自动测试验证
└──────┬──────┘
       │
       ▼
┌─────────────┐
│  5. 迭代    │ ← 循环改进
└─────────────┘
```

---

## 🚀 快速开始

### 前提条件

```bash
# 1. 安装 Ollama
# macOS
brew install ollama

# Windows
# 从 https://ollama.ai 下载安装

# Linux
curl -fsSL https://ollama.ai/install.sh | sh

# 2. 下载模型
ollama pull llama2

# 3. 启动 Ollama
ollama serve
```

### 基本使用

```bash
# 运行 3 次迭代
python karpov_cycle.py --iterations 3

# 使用不同模型
python karpov_cycle.py --model codellama --iterations 5

# 仅分析代码
python karpov_cycle.py --analyze-only

# 自动应用改进 (实验性)
python karpov_cycle.py --auto-apply --iterations 3
```

---

## 🔧 命令行参数

| 参数 | 说明 | 默认值 | 示例 |
|------|------|--------|------|
| `-i, --iterations` | 迭代次数 | 3 | `-i 5` |
| `-m, --model` | Ollama 模型 | llama2 | `-m codellama` |
| `-u, --url` | Ollama API 地址 | localhost:11434 | `-u http://localhost:11434` |
| `-a, --auto-apply` | 自动应用改进 | False | `-a` |
| `--analyze-only` | 仅分析 | False | `--analyze-only` |

## 📊 系统架构

### 核心组件

```
karpov_cycle.py
│
├── OllamaClient          # Ollama API 客户端
│   ├── chat()            # 聊天接口
│   └── generate()        # 生成接口
│
├── CodeAnalyzer          # 代码分析器
│   ├── get_file_stats()  # 文件统计
│   ├── analyze_code_quality()  # 质量分析
│   └── get_code_context()      # 代码上下文
│
├── TestValidator         # 测试验证器
│   └── run_tests()       # 运行测试
│
└── KarpovCycle           # 卡帕斯循环
    ├── run_cycle()       # 运行一次循环
    └── run_iterations()  # 运行多次迭代
```

## 💡 最佳实践

### 1. 选择合适的模型

```bash
# 通用建议
ollama pull llama2
python karpov_cycle.py --model llama2

# 代码改进
ollama pull codellama
python karpov_cycle.py --model codellama

# 高质量建议
ollama pull mistral
python karpov_cycle.py --model mistral
```

### 2. 设置合理的迭代次数

```bash
# 快速检查：1-2 次
python karpov_cycle.py -i 2

# 深度改进：5-10 次
python karpov_cycle.py -i 5

# 过度迭代可能导致：
# - 改进建议重复
# - 时间成本增加
```

### 3. 人工审核改进建议

```bash
# 1. 运行分析
python karpov_cycle.py --analyze-only

# 2. 查看建议
cat karpov_report.md

# 3. 手动应用有价值的建议

# 4. 运行测试验证
python karpov_cycle.py --analyze-only
```

### 4. 结合 Git 使用

```bash
# 1. 创建新分支
git checkout -b karpov-improvements

# 2. 运行卡帕斯循环
python karpov_cycle.py -i 3

# 3. 查看改进
git diff

# 4. 选择性应用
git add -p
git commit -m "feat: 应用卡帕斯循环改进"
```
