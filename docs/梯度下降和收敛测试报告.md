## 3. 损失函数分析

### 损失组成

```
总损失 = MSE 损失 + 0.1 × 稳定性正则化

其中:
- MSE 损失：预测值与真实值的均方误差
- 稳定性正则化：||dz/dt||² (防止动力学爆炸)
- 权重比：MSE : Stability = 1.0 : 0.1
```

### 各组件贡献

| 组件 | 权重 | 作用 |
|------|------|------|
| MSE Loss | 1.0 | 主要任务损失 |
| Stability Loss | 0.1 | 正则化，防止动力学不稳定 |

### 稳定性机制

```python
# 1. τ 限幅
tau = torch.clamp(tau, tau_min=0.01, tau_max=1.0)

# 2. dz/dt 归一化
dzdt = torch.clamp(dzdt, -dzdt_max, dzdt_max)

# 3. z 状态限幅
z = torch.clamp(z, -z_max, z_max)

# 4. 梯度裁剪
grad_norm = clip_grad_norm_(max_norm=1.0)

# 5. 稳定性正则化
loss = mse_loss + 0.1 * ||dz/dt||²
```

## 7. 问题诊断与建议

### 如果遇到问题...

#### 问题 1: 梯度爆炸

**症状**: Loss 突然变为 NaN 或 Inf

**解决方案**:
```python
# 1. 降低学习率
optimizer = Adam(model.parameters(), lr=5e-3)

# 2. 更严格的梯度裁剪
clip_grad_norm_(max_norm=0.5)

# 3. 增加稳定性权重
loss = mse_loss + 0.5 * stability_loss
```

#### 问题 2: 梯度消失

**症状**: 梯度接近零，损失不下降

**解决方案**:
```python
# 1. 增大学习率
optimizer = Adam(model.parameters(), lr=5e-2)

# 2. 减小 dt
model = TwistorLNN(..., dt=0.01)

# 3. 减小稳定性权重
loss = mse_loss + 0.01 * stability_loss
```

#### 问题 3: 不收敛

**症状**: 损失在一定轮次后不再下降

**解决方案**:
```python
# 1. 增加 hidden_dim
model = TwistorLNN(..., hidden_dim=64)

# 2. 增加训练轮次
n_epochs = 200

# 3. 使用学习率调度器
scheduler = ReduceLROnPlateau(patience=10, factor=0.5)
```
