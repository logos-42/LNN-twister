# 一、核心数学模型（你实现的“真内核”）

最常用的是 **LTC（Liquid Time-Constant）模型**：

\tau(h) \frac{dh(t)}{dt} = -h(t) + W,\sigma(h(t)) + U,x(t) + b

### 拆解（工程对应）

* ( h(t) )：隐藏状态（连续时间）
* ( x(t) )：输入
* ( \tau(h) )：时间常数（关键：可学习）
* ( W, U, b )：参数
* ( \sigma )：非线性（tanh / sigmoid）

👉 变形一下方便写代码：

[
\frac{dh}{dt} = \frac{-h + W\sigma(h) + Ux + b}{\tau(h)}
]

## Step 1：定义动力系统（核心模块）

```python
import torch
import torch.nn as nn

class LTCCell(nn.Module):
    def __init__(self, input_size, hidden_size):
        super().__init__()
        self.hidden_size = hidden_size

        # 状态更新
        self.W_h = nn.Linear(hidden_size, hidden_size)
        self.W_x = nn.Linear(input_size, hidden_size)

        # 时间常数 τ(h)
        self.W_tau = nn.Linear(hidden_size, hidden_size)

    def forward(self, t, h, x):
        # tau(h)
        tau = torch.sigmoid(self.W_tau(h)) + 1e-3

        # dynamics
        dh = (-h + torch.tanh(self.W_h(h)) + self.W_x(x)) / tau

        return dh
```

## Step 2：ODE求解器封装

### 方法A：手写 Euler（最简单，可跑）

```python
def euler_step(cell, h, x, dt):
    dh = cell(None, h, x)
    return h + dt * dh
```

### 方法B：使用 ODE Solver（推荐）

```python
from torchdiffeq import odeint

class ODEFunc(nn.Module):
    def __init__(self, cell, x):
        super().__init__()
        self.cell = cell
        self.x = x

    def forward(self, t, h):
        return self.cell(t, h, self.x)
```

## Step 3：时间序列 forward（关键结构）

```python
class LiquidNet(nn.Module):
    def __init__(self, input_size, hidden_size, output_size):
        super().__init__()
        self.cell = LTCCell(input_size, hidden_size)
        self.output = nn.Linear(hidden_size, output_size)

    def forward(self, x_seq):
        # x_seq: [T, B, input_size]
        T, B, _ = x_seq.shape

        h = torch.zeros(B, self.cell.hidden_size).to(x_seq.device)

        outputs = []

        for t in range(T):
            x = x_seq[t]

            # Euler
            h = h + 0.1 * self.cell(None, h, x)

            outputs.append(self.output(h))

        return torch.stack(outputs)
```

# 四、你必须注意的坑（否则你会卡死）

我直接给你踩坑经验：

---

## ⚠️ 1. τ(h) 不能太小

```python
tau = sigmoid(...) + 1e-3
```

否则：

👉 数值爆炸（除以接近0）

---

## ⚠️ 2. dt（时间步长）

* 太大 → 不稳定
* 太小 → 慢

建议：

```python
dt = 0.05 ~ 0.1
```

---

## ⚠️ 3. 梯度问题

如果你用 ODE：

```python
from torchdiffeq import odeint_adjoint
```

👉 节省内存，但有数值误差

---

## ⚠️ 4. 输入是连续时间（关键优势）

你可以做：

```python
x(t) 不等间隔
```

这是 LNN 比 Transformer 强的地方
