# Twistor-LNN 零样本和多任务学习扩展文档

**更新日期**: 2026-03-28  
**版本**: v2.0 (支持零样本和多任务学习)

---

## 📋 概述

本次更新为 Twistor-LNN 添加了以下能力：

1. **多任务学习** - 单个模型处理多个相关任务
2. **零样本学习** - 在未见过的新任务上直接推理
3. **元学习适应** - 使用少量样本快速适应新任务
4. **提示学习** - 基于提示向量实现任务迁移

---

## 🎯 三种扩展方案对比

| 方案 | 类名 | 适用场景 | 零样本能力 | 多任务能力 | 训练复杂度 |
|------|------|---------|-----------|-----------|-----------|
| **多任务 Twistor** | `MultiTaskTwistorLNN` | 多个相关任务 | ⭐⭐⭐ | ⭐⭐⭐⭐⭐ | 中等 |
| **元学习 Twistor** | `MetaTwistorLNN` | 快速适应新任务 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐ | 高 |
| **提示学习 Twistor** | `PromptTwistorLNN` | 任务组合迁移 | ⭐⭐⭐⭐ | ⭐⭐⭐⭐ | 低 |

## 方案 1: 多任务 Twistor-LNN

### 核心架构

```
┌─────────────────────────────────────────┐
│           Task Embedding                │
│         (每个任务一个向量)                │
└─────────────────┬───────────────────────┘
                  │
    ┌─────────────┼─────────────┐
    │             │             │
    ▼             ▼             ▼
Input Proj   Dynamics Core  Output Proj
(任务特定)    (共享)         (任务特定)
```

### 使用示例

```python
from twistor_lnn_multitask import MultiTaskTwistorLNN, TaskConfig

# 1. 定义多个任务
task_configs = [
    TaskConfig(task_id=0, task_name="sine", input_dim=2, output_dim=1),
    TaskConfig(task_id=1, task_name="cosine", input_dim=2, output_dim=1),
    TaskConfig(task_id=2, task_name="lorenz", input_dim=3, output_dim=3),
]

# 2. 创建模型
model = MultiTaskTwistorLNN(
    task_configs=task_configs,
    hidden_dim=32,
    task_embedding_dim=8,
    dt=0.1
)

# 3. 多任务训练
for epoch in range(n_epochs):
    # 任务 1: 正弦波
    x_sine = torch.randn(20, 4, 2)
    y_sine = torch.randn(20, 4, 1)
    y_pred_sine = model(x_sine, task_name="sine")
    loss_sine = F.mse_loss(y_pred_sine, y_sine)
    
    # 任务 2: 余弦波
    x_cos = torch.randn(20, 4, 2)
    y_cos = torch.randn(20, 4, 1)
    y_pred_cos = model(x_cos, task_name="cosine")
    loss_cos = F.mse_loss(y_pred_cos, y_cos)
    
    # 联合优化
    total_loss = loss_sine + loss_cos
    total_loss.backward()
    optimizer.step()

# 4. 零样本迁移 (使用源任务的输入投影 + 目标任务的输出投影)
y_pred_transfer = model.zero_shot_transfer(
    x_test, 
    source_task="sine", 
    target_task="cosine"
)
```

### 关键特性

- **任务嵌入**: 每个任务有一个可学习的嵌入向量
- **共享动力学**: 所有任务共享核心动力学参数
- **任务特定投影**: 输入/输出层针对每个任务特定
- **门控机制**: 动态调整动力学参数

## 方案 2: 元学习 Twistor-LNN (MAML)

### 核心架构

```
MAML 两阶段训练:

1. 内循环 (适应):
   在支持集上梯度下降
   θ' = θ - α∇L_support(θ)

2. 外循环 (元更新):
   在查询集上评估
   更新 θ 以最小化 L_query(θ')
```

### 使用示例

```python
from twistor_lnn_multitask import MetaTwistorLNN

# 1. 创建元学习模型
meta_model = MetaTwistorLNN(
    input_dim=2,
    hidden_dim=32,
    output_dim=1,
    dt=0.1
)

# 2. 元训练 (MAML)
for epoch in range(n_meta_epochs):
    # 采样一批任务
    tasks = sample_tasks(batch_size=4)
    
    meta_loss = 0
    for task in tasks:
        x_support, y_support = task.support_set
        x_query, y_query = task.query_set
        
        # MAML 元更新
        query_loss, _ = meta_model.meta_update(
            x_support, y_support,
            x_query, y_query,
            inner_lr=0.1,
            inner_steps=5
        )
        meta_loss += query_loss
    
    # 更新元参数
    meta_loss.backward()
    optimizer.step()

# 3. 零样本适应新任务 (仅需少量样本)
x_few_shot = torch.randn(5, 2, 2)  # 5 个样本
y_few_shot = torch.randn(5, 2, 1)
x_test = torch.randn(10, 2, 2)

# 快速适应 (5 步梯度下降)
y_pred = meta_model.zero_shot_adapt(
    x_few_shot, y_few_shot, x_test,
    adapt_steps=5,
    adapt_lr=0.1
)
```

### 关键特性

- **元初始化**: 学习一个好的参数初始化
- **快速适应**: 仅需几步梯度下降即可适应新任务
- **少样本学习**: 5-10 个样本即可达到可接受性能

## 方案 3: 提示学习 Twistor-LNN

### 核心架构

```
┌─────────────────────────────────────────┐
│         Prompt Bank (10 个提示向量)      │
└─────────────────┬───────────────────────┘
                  │
    ┌─────────────▼─────────────┐
    │   Prompt Selector         │
    │   (根据输入选择提示)       │
    └─────────────┬─────────────┘
                  │
    ┌─────────────▼─────────────┐
    │   Prompt Projection       │
    │   (注入到动力学)          │
    └───────────────────────────┘
```

### 使用示例

```python
from twistor_lnn_multitask import PromptTwistorLNN

# 1. 创建提示学习模型
prompt_model = PromptTwistorLNN(
    input_dim=2,
    hidden_dim=32,
    output_dim=1,
    n_prompts=10,  # 10 个提示向量
    prompt_dim=8,
    dt=0.1
)

# 2. 训练 (自动学习提示选择和组合)
for epoch in range(n_epochs):
    x = torch.randn(20, 4, 2)
    y = torch.randn(20, 4, 1)
    
    y_pred = prompt_model(x)
    loss = F.mse_loss(y_pred, y)
    
    loss.backward()
    optimizer.step()

# 3. 零样本迁移
# 提示模型自动根据输入选择合适提示
x_new_task = torch.randn(20, 4, 2)
y_pred_new = prompt_model(x_new_task)  # 直接推理
```

### 关键特性

- **提示库**: 学习一组通用提示向量
- **自动选择**: 根据输入动态选择提示组合
- **零样本迁移**: 新任务自动匹配合适提示

## 🎯 选择指南

### 选择多任务学习，如果:
- ✅ 有多个相关任务需要同时处理
- ✅ 任务之间有正向迁移潜力
- ✅ 需要共享表示提高效率

### 选择元学习，如果:
- ✅ 需要快速适应新任务
- ✅ 新任务只有少量样本
- ✅ 可以接受较高的训练成本

### 选择提示学习，如果:
- ✅ 需要零样本迁移能力
- ✅ 任务可以用提示组合描述
- ✅ 需要低训练成本的方案


## 🔧 故障排除

### 问题 1: 多任务训练不收敛

**解决方案**:
```python
# 1. 增加任务嵌入维度
task_embedding_dim = 16  # 从 8 增加到 16

# 2. 使用梯度累积
accum_steps = 4
for i, (x, y) in enumerate(dataloader):
    loss = compute_loss(x, y)
    if i % accum_steps == 0:
        optimizer.zero_grad()
    (loss / accum_steps).backward()
```

### 问题 2: 元学习 OOM

**解决方案**:
```python
# 1. 减少内层步数
inner_steps = 3  # 从 5 减少到 3

# 2. 使用一阶近似 (FOMAML)
# 在 meta_update 中设置 create_graph=False
```

### 问题 3: 提示学习效果差

**解决方案**:
```python
# 1. 增加提示数量
n_prompts = 30  # 从 10 增加到 30

# 2. 使用更复杂的选择网络
self.prompt_selector = nn.Sequential(
    nn.Linear(input_dim, 32),
    nn.ReLU(),
    nn.Linear(32, 16),
    nn.ReLU(),
    nn.Linear(16, n_prompts),
)
```
